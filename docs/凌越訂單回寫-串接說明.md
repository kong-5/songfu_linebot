# 凌越訂單回寫（雲端 ↔ 內網）串接說明

把後台訂單「回寫／匯入」凌越 ERP，並把凌越配發的**單據號碼**顯示回後台。

## 為什麼要分兩段

凌越是**本機 LAN 版** SOAP API（`http://192.168.4.11/erpapi/erpservice.svc`，私有 IP、不走外網）。
後台跑在 **Cloud Run（外網）**，連不到 192.168.4.x 內網，所以：

```
Cloud Run 後台（出資料 / 收單據號）  ←HTTPS→  內網 agent（Dispatch 那台）  ←LAN→  凌越 LyDataIn
```

- 雲端只負責「**輸出待回寫訂單**」與「**接收凌越單據號並顯示**」。
- 真正呼叫凌越寫入 API（`LyGetPassKey` → 寫入 method）的動作，由**內網那台已在跑每日排程的機器**執行。
- **單據號順編**：凌越端／agent 依該日既有單據往後配號，雲端不指定 DocNo，避免撞號。

## 雲端提供的兩支 API（機器對機器）

認證：HTTP 標頭 `X-Writeback-Key`，需等於 Cloud Run 環境變數 `LINGYUE_WRITEBACK_KEY`。
未設定該環境變數時，端點回 `503`（等於停用）；金鑰不符回 `401`。

### 1) 拉「待回寫」訂單

```
GET /admin/lingyue-writeback/pending?date=YYYY-MM-DD
X-Writeback-Key: <金鑰>
```

`date` 省略時預設今天（台北）。只回該日、未刪除、且**尚未回寫**（`lingyue_written_at` 為空）、有可匯出明細的訂單。

回傳範例：

```json
{
  "date": "2026-06-06",
  "count": 1,
  "orders": [
    {
      "order_id": "o_xxx",
      "order_no": "A001",
      "order_date": "2026/06/06",
      "customer_code": "LY001",
      "customer_name": "測試客戶",
      "doc_remark": "",
      "items": [
        { "product_code": "PLU99", "product_name": "高麗菜", "unit": "公斤", "quantity": 10, "item_note": "" }
      ]
    }
  ]
}
```

欄位對應凌越 Excel 匯出邏輯：
`customer_code` = 凌越編號（HQCustCode，無則退寺岡編號）、`product_code` = 凌越料號（erp_code）。
**agent 端再把這些欄位對映到凌越「寫入 method」的實際參數**（參數名稱待跟廠商確認後在 agent 端補上）。

### 2) 回填凌越單據號

```
POST /admin/lingyue-writeback/callback
X-Writeback-Key: <金鑰>
Content-Type: application/json

{ "results": [ { "order_id": "o_xxx", "doc_no": "LY-2026-0016", "ok": true } ] }
```

- `ok: false` 或缺 `doc_no` 的項目視為失敗，**不會**標記為已回寫（下次 `pending` 會再出現，可重試）。
- 成功者寫入 `orders.lingyue_doc_no` 並記錄 `lingyue_written_at`。

回傳：`{ "updated_count": 1, "failed_count": 0, "updated": [...], "failed": [...] }`

## 後台顯示

- 訂單列表：回寫後出現 `↩單據號` 標記（青色）。
- 訂單明細：標題列顯示「凌越單據號 …」。

## 內網 agent 的工作流程（給 Dispatch 的指令骨架）

```
每日（或手動）執行：
1. GET https://<後台網址>/admin/lingyue-writeback/pending?date=今天
   帶標頭 X-Writeback-Key: <金鑰>
2. 對每張 order：
   a. LyGetPassKey(pusid, pverifykey) 取 30 秒金鑰 ikye（失效 -3 自動重拿重試）
   b. 把 order 欄位組成凌越「寫入 method」需要的 XML/物件
      （DocNo 不要自己亂編，依凌越該日既有單據順編或讓凌越配發）
   c. 呼叫寫入 method，取回凌越配發的單據號
3. POST https://<後台網址>/admin/lingyue-writeback/callback
   body: { results: [ { order_id, doc_no, ok, error } , ... ] }
注意：過程中不要動到任何正式既有單據；先用一筆測試單驗證再批次。
```

## 設定

Cloud Run 環境變數新增：

```
LINGYUE_WRITEBACK_KEY=<自訂一組長亂碼，內網 agent 與後台共用>
```

未設定時兩支端點都停用（回 503），不影響其他後台功能。
