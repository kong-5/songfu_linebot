# 凌越整合代理（視窗程式）

把原本要**分開跑的兩支內網代理**「包成一支有視窗的程式」，掛在能連凌越 LAN 的那台
Windows（`D:\Work\lystk_tool` 旁）上常駐執行。一個視窗就能看到現在的連線狀態。

## 合併了哪兩支

| 原本 | 原本怎麼跑 | 現在（整合視窗） |
|------|-----------|------------------|
| **庫存代理** `ly_stock_agent.py`（用 `ly_stock_push.py`） | 雙擊 `ly_stock_agent_run.bat`，黑視窗常駐 | 視窗內背景執行緒：長連線即時推 ＋ 每日定時推 |
| **訂單回寫** `ly_writeback_bridge.py`（用 `ly_order`） | 工作排程器每天叫 `ly_writeback_daily.bat` | 視窗內背景執行緒：長連線等 `/wait`，**只寫你在網站按過『上傳凌越』的單**（見 `凌越回寫-規則與必備設定.md`） |

> 原本兩支腳本**仍保留**，可獨立使用；整合視窗是「多一個把兩者包在一起、看得到狀態」的選項。

## 視窗會顯示什麼

- ● **雲端連線**：是否連得上後台、最後連上的時間。
- ● **凌越模組**：`lystk` / `ly_order` 有沒有載入成功（此機是否有凌越環境）。
- ● **庫存代理**：運作中／最後推送時間／下次定時。
- ● **訂單回寫**：自動回寫開關／最後回寫時間與結果。
- **即時訊息記錄**：原本黑視窗的所有輸出都會顯示在這一塊。
- **手動按鈕**：立即推庫存、回寫試跑(dry-run)、回寫測一張、立即回寫。
- **⚙ 設定**：網址／金鑰／公司別／定時／回寫間隔／倉別／單價，存成 `ly_agent_config.json`。

## 怎麼用

### 做法 A：那台已裝 Python
1. 雙擊 `scripts/ly_agent_gui_run.bat`（用 `pythonw` 開，無黑色 console）。
2. 第一次打開按「⚙ 設定」，填好**雲端後台網址**與**回寫金鑰**（與後台 `LINGYUE_WRITEBACK_KEY` 相同），按儲存。
3. 按「啟動代理」。之後每次打開會依設定自動啟動。

### 做法 B：打包成單一 .exe（該台可不裝 Python）
1. 在已裝 Python 的機器執行 `scripts/build_agent_exe.bat`（會自動裝 PyInstaller 並打包）。
2. 產生 `scripts/dist/凌越整合代理.exe`，複製到內網那台，雙擊即可。
3. exe 執行時用環境變數 `LYSTK_DIR`（預設 `D:\Work\lystk_tool`）去找凌越模組；
   設定檔 `ly_agent_config.json` 會存在 exe 同資料夾。

### 開機自動跑
工作排程器 → 觸發程序「登入時」→ 動作選 `ly_agent_gui_run.bat` 或 exe →
「開始位置」填 `D:\Work\lystk_tool`（確保找得到 `lystk.py` / `ly_order.py`）。

## 設定項目（＝環境變數對照）

| 視窗欄位 | 對應環境變數 | 說明 |
|----------|--------------|------|
| 雲端後台網址 | `LY_CLOUD_BASE` | 例 `https://xxxx.run.app`（必填） |
| 回寫金鑰 | `LINGYUE_WRITEBACK_KEY` | 與後台相同那把（必填） |
| 公司代碼 | `LY_ICPNO` | 松富=00、龍港=01、桂田=03 |
| 每日定時推送 | `LY_STOCK_TIMES` | 如 `06:00,12:00`；清空＝只靠按鈕/即時 |
| 掛著自動處理『上傳凌越』佇列 | — | 勾選＝只寫你在網站按過「上傳凌越」的單（長連線 `/wait`），不會動其他訂單 |
| 預設倉別 | `LY_DEFAULT_WHNO` | **必填**，否則匯入的訂單沒有倉別 |
| 預設倉別 | `LY_DEFAULT_WHNO` | 留空＝之後在凌越補倉別 |
| 預設單價 | `LY_DEFAULT_PRICE` | 留空＝讓凌越依客戶售價表帶價 |

> 設定優先序：視窗設定檔 > 環境變數 > 預設值。填好一次就不必再改 `.bat`。

## 安全提醒（訂單回寫）

回寫會**實際寫進凌越訂貨單**。上線前建議先用視窗按鈕「回寫測一張」跑一次
（寫第一張標記【API測試請刪除】→ 回查 → 自動刪除、不回填後台），確認凌越收得進去，
再開「掛著自動處理」。訂單回寫**只寫你在網站按過『上傳凌越』的單**，不會動其他訂單。

## 疑難排解

### 一直「雲端連線問題：Not Found（404）」
先按視窗上的 **「測試連線」**（用 `pending` 端點探測，能分辨是哪種問題）：

- **測試顯示連線成功、金鑰正確** → 表示網址/金鑰沒問題，是**後台缺庫存端點**：
  你線上的後台版本較舊，還沒有 `inventory-wait` / `inventory-push`（庫存那組是後來才加）。
  程式會自動切成 **「定時模式」**（不再長輪詢），庫存的即時刷新要等**後台更新到最新版**才會通。
  訂單回寫不受影響、照常運作。
- **測試顯示 HTTP 401** → 金鑰不對，按「⚙ 設定」改成與後台 `LINGYUE_WRITEBACK_KEY` 相同那把。
- **測試顯示 HTTP 404 / 連不到** → 網址填錯，或此機連不到該網址；確認「⚙ 設定」的網址。

### 回寫失敗：`'types.SimpleNamespace' object has no attribute ...`
表示載入到**舊版** `ly_writeback_bridge.py`（多半是 `D:\Work\lystk_tool` 裡的舊檔）。
本程式已改成「**隨附版本優先**」，並把正確的 `ly_stock_push.py` / `ly_writeback_bridge.py`
一起放在同一個資料夾。**請確保這三個檔案放在一起**（解壓後的資料夾就是），
不要只單獨複製 `ly_agent_gui.py`。

### 一次發現很多張待回寫訂單
第一次啟動若看到「發現 N 張待回寫訂單」，代表雲端累積了待回寫的單。
**上線前務必先對一張訂單在網站點「上傳凌越」，再按視窗「檢查上傳佇列(試跑)」→「立即處理上傳佇列」**確認凌越收得進去（倉別有、單位 KG、料號對），再放心讓它掛著。

## 需要的環境

Windows ＋ Python（含內建 tkinter，Windows 版預設就有）＋ `D:\Work\lystk_tool` 內的
`lystk.py` / `ly_order.py`（跟原本兩支代理相同的依賴）。
