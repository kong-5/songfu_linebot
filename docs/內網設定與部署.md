# 內網設定與部署（凌越那台 Windows）

> 內網機器上要放什麼、怎麼設、怎麼常駐。程式規則見 `資料處理規則.md`。
> 雲端後台部署見 `推Git就部署-設定.md`／`做法二-Cloud-SQL-詳細步驟.md`。

---

## 1. 架構一張圖

```
雲端 Cloud Run 後台（外網）
   │  HTTPS（X-Writeback-Key 認證；長連線 /wait、/inventory-wait）
   ▼
內網那台 Windows（Dispatch 那台，D:\Work\lystk_tool 旁）
   │  LAN SOAP（http://192.168.4.11/erpapi/erpservice.svc）
   ▼
凌越 ERP（LAN 版，外網連不到）
```

凌越只在公司內網，雲端打不進去 → 所有讀寫凌越的程式**都跑在這台 Windows**，
由它主動連雲端（長連線等待），所以不用開防火牆入站。

## 2. 這台機器的必備條件

| 項目 | 說明 |
|---|---|
| 作業系統 | Windows（可連凌越 LAN） |
| Python | 已安裝（含內建 tkinter）；套件：`zeep`（lystk 用）、選配 `openpyxl`（匯 Excel） |
| `D:\Work\lystk_tool\` | Dispatch 維護的凌越底層模組：`lystk.py`、`ly_order.py`、`ly_datain.py` |
| 本專案 scripts | 放在任意資料夾（建議 `D:\Work\lystk_tool` 旁），**同資料夾整包放**，不要單抽一支 |

程式尋找凌越模組的路徑：環境變數 `LYSTK_DIR`（預設 `D:\Work\lystk_tool`）。

## 4. 環境變數（＝視窗⚙設定的對照）

| 環境變數 | 必填 | 預設 | 說明 |
|---|:--:|---|---|
| `LY_CLOUD_BASE` | ✅ | — | 雲端後台網址 `https://xxxx.run.app` |
| `LY_WRITEBACK_KEY` | ✅ | — | ＝後台 `LINGYUE_WRITEBACK_KEY` |
| `LY_ICPNO` | | `00` | 公司代碼（00 松富、01 龍港、03 桂田） |
| `LY_DEFAULT_WHNO` | 建議 | 空 | 後備倉別（品項查不到 SK_RKWHNO 時補） |
| `LY_DEFAULT_PRICE` | | 空 | 留空＝凌越依售價表帶價 |
| `LY_CREATE_NAME` | | `052` | 建立人/製單人操作員代碼 |
| `LY_MAKER` | | 空 | 覆寫製單人（通常不用） |
| `LY_AUDITED` | | `1` | 1=寫入即已審核（拋轉需要）；0=未審核 |
| `LY_STOCK_TIMES` | | `06:00,12:00` | 庫存每日定時推送 |
| `LYSTK_DIR` | | `D:\Work\lystk_tool` | 凌越模組位置 |
| `LY_QUERY_KEY` | 用查詢服務才要 | — | `ly_query_server.py` 的金鑰 |

> **視窗版免設環境變數**：⚙ 設定存 `ly_agent_config.json`（在程式同資料夾），
> 優先序＝設定檔 > 環境變數 > 預設值。

## 5. 安裝與常駐（視窗版，建議做法）

1. 把整包 scripts（至少 `ly_agent_gui.py`＋`ly_writeback_bridge.py`＋`ly_stock_push.py`
   ＋`ly_agent_gui_run.bat`）放同一資料夾。
2. 雙擊 `ly_agent_gui_run.bat`（用 `pythonw` 開、無黑視窗）。
3. ⚙ 設定填：網址、金鑰、公司別、後備倉別、建立人代碼 → 「測試連線」→ 儲存 → 「啟動代理」。
4. **開機自動跑**：工作排程器 → 觸發「登入時」→ 動作選該 `.bat` →
   「開始位置」填 `D:\Work\lystk_tool`。
5. 要單一 exe：跑 `build_agent_exe.bat`（PyInstaller），產物在 `dist\凌越整合代理.exe`。

上線前 SOP（一次就好）：
網站對一張單按「上傳凌越」→ 視窗「檢查上傳佇列(試跑)」看倉別/單位/欄位
→「立即處理上傳佇列」→ 進凌越確認：倉別有、單位 KG、付款方式/業務員有、已審核。

## 6. 雲端後台的對應設定（Cloud Run 環境變數）

| 變數 | 說明 |
|---|---|
| `LINGYUE_WRITEBACK_KEY` | 與內網 `LY_WRITEBACK_KEY` 同一把長亂碼；未設＝機器端點全部停用(503) |

後台缺 `/wait`／`/inventory-wait` 等端點（agent 顯示 404）＝ 後台版本舊 → 重新部署 main。

## 7. 已知陷阱（都踩過了，別再踩）

1. **lystk 無逾時會卡死** → 一律注入 `Transport(timeout=60)` 的 zeep client（各程式已內建）。
2. **舊版模組蓋新版** → 程式以「同資料夾隨附版優先」絕對路徑載入；整包放一起就不會錯。
3. **常駐回寫接 `/pending` 會把整天訂單全寫進去** → 只准接 `/wait`（規則見資料處理規則 §1.1）。
4. **空倉別/中文單位/未審核/缺建立日期 → 拋轉失敗** → `map_order` 已自動補齊，不要繞過它自己組欄位。
5. **重轉不會刪凌越舊單** → 先進凌越刪舊單再按「重新轉入」。
