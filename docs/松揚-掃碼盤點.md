# 松揚（icpno=02）掃碼盤點 — 手機當 PDA

最後更新：2026-07-10

## 一句話

松揚的冷凍貨/雜貨用**手機掃條碼**盤點：LIFF 頁 `/liff/scan` 連續掃描→帶出品項→輸入數量→送出，
資料寫進**與松富同一套**盤點表（`stocktake_session`/`stocktake_count`），後台「每日盤點」直接看得到
（盤差/比例/CSV 全共用，倉庫卡片帶「松揚」標示）。銷貨照舊走凌越＋學校採購網站；庫存基準由凌越倒出。

## 多公司架構（這次的底層改動）

- 松揚在**同一套凌越**裡，公司代碼 `02`（松富 00、龍港 01、松成 03）。
- `erp_stock_items`、`erp_warehouse` 主鍵改為 **(icpno, 料號/倉號)**——兩公司料號/倉號可重複。
  DB init 內建冪等遷移（SQLite 重建表、PG 換主鍵），舊資料自動補 `icpno='00'`。
- `stocktake_session` 加 `icpno` 欄（NULL 視為 `00`）。「一倉一日一筆」的唯一性含公司。
- 庫存推送端點 `/admin/lingyue-writeback/inventory-push` 改**按公司覆蓋**（只刪該 icpno 的舊列）；
  快照時間另記 `erp_stock_snapshot_at_<icpno>`（legacy 鍵保留＝最後一次推送）。
- 公司代碼→名稱的權威 helper：`dist/lib/erp-companies.js`（`ERP_COMPANY_NAMES`/`erpCompanyName`/`normIcpno`）。

## 內網代理設定（一步）

「凌越整合代理」設定裡的 **公司代碼 (LY_ICPNO)** 改填 **`all`** → 庫存推送**全公司 00,01,02,03 逐家撈、逐家推**
（雲端按公司覆蓋、各自分開）。也可逗號指定（如 `00,02`）。
- `ly_stock_push.py` 的 `push_once` 自己會拆（`all`/留空＝00,01,02,03）；
  **訂單回寫/單品查詢只用第一家非 all 的代碼（`all`＝00 松富）**，不受影響（`ly_agent_gui.py` 的 `first_icpno()`）。
- 記得把新的 `ly_stock_push.py` 放到「凌越整合代理」資料夾（exe 同層）讓 `local_import` 撿走。

## 條碼從零建置（重點流程）

凌越目前**沒有維護商品條碼**，所以條碼對照存在我們自己的 `product_barcode` 表
（`(icpno, barcode)` 主鍵 → 料號＋`qty_per_scan`）：

1. 第一輪盤點＝條碼建檔：掃到**未知條碼**→頁面跳「綁定」→搜品名→選品項→（整箱條碼填每掃單位數）→綁定並帶入。
2. 綁過一次，之後全公司都認得（掃碼頁一進倉就把整家公司的條碼對照載到手機，掃描零延遲）。
3. 一個品項可綁多條碼（單支＋整箱；整箱掃一下＝加一箱的量）。
4. 後台總管理：**庫存管理 → 條碼對照**（`/admin/inventory/barcodes`，手動新增/改倍數/刪除；新增時料號會驗證存在）。

## LIFF 掃碼頁 `/liff/scan`（PDA 操作）

- 進頁**選公司**（膠囊切換 松揚/松富/龍港/松成，記住上次選擇）→ 選倉庫 → 相機**連續掃描**
  （掃到嗶聲＋震動）→ 品項卡＋**PDA 大數字鍵盤**（掃完直接打數字＝覆蓋、續打＝接著輸入、
  ⌫/清/±1/完成；不彈系統鍵盤）→ 掃下一個 → 「送出盤點」。
- 同條碼連掃自動累加（間隔 1.5 秒防重複觸發）；已盤清單點品項可回頭改（自動開鍵盤）。
- **續盤**：同倉同日重開會帶回已送出的數；未送出的草稿存 localStorage，中斷重開自動接回。
- 手動輸入欄：相機掃不到、外接藍牙掃描槍、或直接打料號都走這裡。
- **掃描引擎鏈（iPhone 有支援）**：`BarcodeDetector`（Android 原生）→ **zxing 純 JS 解碼**
  （本地 vendor `dist/liff/vendor/zxing.min.js`，路由 `/liff/vendor/zxing.min.js`；iPhone 的 LINE
  瀏覽器走這條，iOS 14.3+ 可用相機）→ `liff.scanCodeV2()`（按一下掃一個；LIFF app 要開 scanQR）
  → 手動輸入。zxing 版本更新：`npm pack @zxing/library` 取 `umd/index.min.js` 覆蓋 vendor 檔。
- ?icpno= 可指定公司（預設 `02` 松揚），一般直接用頁內切換即可。
- **上線前要做**：LINE Developers 開一個新 LIFF app，端點 `https://<網域>/liff/scan`、**開啟 scanQR**，
  把 LIFF ID 設到 Cloud Run 環境變數 `LIFF_ID_SCAN`（沒設會 fallback 用盤點頁的 LIFF ID，
  在 LINE 內開直連網址可用，但外部瀏覽器登入會失敗）。入口網址：`https://liff.line.me/<LIFF_ID_SCAN>`。

### LIFF API（`dist/liff/index.js`）

| 端點 | 用途 |
|---|---|
| `GET /liff/api/stocktake/warehouses?icpno=` | 倉庫清單（原松富 API 加 icpno，預設 00 行為不變） |
| `GET /liff/api/stocktake/items?warehouse=&icpno=` | 倉內品項＋今日已盤（續盤） |
| `POST /liff/api/stocktake/submit`（body 加 `icpno`） | 送出（同公司同倉同日覆蓋） |
| `GET /liff/api/scan/barcodes?icpno=` | 整家公司條碼對照（一次載入） |
| `GET /liff/api/scan/search?icpno=&q=` | 綁定時搜品項（全公司不限倉） |
| `POST /liff/api/scan/bind` | 綁定條碼（記綁定者 LINE 名字） |

## 後台（全部共用既有入口）

- **目前庫存**：公司切換分段（松富/松揚，有推送過的公司自動出現）；快照時間按公司顯示；
  點品項查進銷存會帶該公司的 icpno 給內網代理。
- **倉庫設定**：按公司分頁；儲存只覆蓋該公司的倉別設定。
- **每日盤點**：松揚倉自動出現（卡片/內頁帶「松揚」藍標）；倉庫連結鍵為 `icpno:倉號`（舊網址 `?wh=倉號` 視為松富）；
  「對最新盤差」按公司比對庫存快照；CSV 兩公司都在。
- **條碼對照**：見上。
