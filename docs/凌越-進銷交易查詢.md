# 庫存管理 — 目前庫存「點品項 → 近期進銷交易」＋ 凌越 API 資料種類權威整理

後台「庫存管理 → 目前庫存」點任一品項，右側抽屜即時查該料號在凌越的**近期進銷交易明細**
（日期／類型／客戶／進出數量／單價／金額），最上面顯示 **Σ入庫 / Σ出庫 / 淨變動**，方便對盤差。
最後更新：2026-07-08

> 架構與「不要重複踩」的權威清單在 repo 根目錄 **`CLAUDE.md`**；本文件為進銷交易查詢細節，
> 並把松富手上那份「凌越 API 串接技術文件」整理包的重點**記錄下來**（欄位／資料種類代碼）。

---

## 1. 這功能查得到什麼（很重要，先看）

凌越 WCF `LyDataOut`（轉出/查詢）**能匯出的資料種類就這幾種**（見下方 §3，抄自官方 API 文件）。
其中和「庫存進出」有關的**只有兩種單別**：

| 資料種類 | 單別 | 對庫存 | 主表前綴 | 明細前綴 |
|---|---|---|---|---|
| `0000A1` | 銷貨單 | **出庫（−）** | `SP_` | `SD_` |
| `0000A2` | 銷貨退回單 | **入庫（＋）** | `SP_` | `SD_`（欄位與銷貨單相同）|

> ⚠ **這個凌越 API 沒有「進貨單」資料種類。** 官方 `LyDataOut` 支援清單只有
> `000000 / 000004 / 000009 / 00000D / 0000A0 / 0000A1 / 0000A2`，沒有進貨單。
> 所以「近期進銷交易」目前＝**銷貨（出）＋ 銷退（入）**。若日後向凌越取得進貨單的
> 資料種類代碼，設環境變數 `LY_PURCHASE_IDAKD`（見 §4）即會一併查、標成入庫(＋)。

因此對盤差時：**淨變動 = Σ銷退(入) − Σ銷貨(出)**。若帳面現有量與盤點對不上，這份明細
可以逐筆核對哪張銷貨／銷退造成差異。

## 2. 架構（雲端連不到凌越 LAN，一律兩段；on-demand 長連線）

```
 內網 Windows「凌越整合代理」ly_agent_gui.py            雲端後台 Cloud Run
 ────────────────────────────────────────            ──────────────────
  _txn_loop 長輪詢 txn-wait ◀── 使用者點品項排入佇列 ── /admin/inventory/stock/txn-request
       │ 撿到料號 → local_import ly_item_txn                       ▲
       │ fetch_item_records(icpno, code)  查 A1(出)+A2(入)          │ 抽屜每 1.5s 輪詢
       └── POST 回填 ──▶ /admin/lingyue-writeback/txn-callback ─▶ 存 app_settings.erp_txn_res_<料號>
                                                    使用者看 ◀── /admin/inventory/stock/txn?code=
```

- 機器端點沿用訂單回寫同一把金鑰 `LINGYUE_WRITEBACK_KEY`（標頭 `X-Writeback-Key`）。
- 結果在雲端快取 `ERP_TXN_FRESH_MS`（新鮮期內再點同一品項直接給快取，不重查凌越）。
- 真正查凌越的程式：**`scripts/ly_item_txn.py`**（原始碼）→ 佈到「凌越整合代理」資料夾，
  由 `ly_agent_gui.py` 用 `local_import` 載入。**更新查詢邏輯＝換該資料夾的 `ly_item_txn.py`**，
  不是 `D:\Work\lystk_tool`（見 CLAUDE.md）。

### 回傳資料格式（ly_item_txn.fetch_item_records → txn-callback → 抽屜）

```jsonc
{
  "records": [
    { "date":"2026-07-01", "kind":"銷貨", "dir":"out", "customer":"○○行",
      "qty":120, "sqty":-120, "price":25, "amount":3000, "doc_no":"SP..." },
    { "date":"2026-06-28", "kind":"銷退", "dir":"in",  "customer":"○○行",
      "qty":10,  "sqty":10,  "price":25, "amount":250,  "doc_no":"SR..." }
  ],
  "count": 2,
  "sum_in": 10, "sum_out": 120, "net": -110,   // 抽屜上方三格：Σ入 / Σ出 / 淨變動
  "note": ""
}
```

- `dir`：`in`＝入庫(＋)、`out`＝出庫(−)。抽屜用它上色、算帶號數量與加總。
- 舊版代理若沒回 `dir`/`sum_*`，抽屜會**用類型字（「退／進／入」→入庫）在前端自算**，不會壞。

## 3. 凌越 API 資料種類代碼（抄自官方「WCF模式-資料彙出入處理說明」）

`LyDataIn`（轉入）與 `LyDataOut`（轉出/查詢）帶 `idakd`（資料種類）指定單別：

| 代碼 | 種類 | LyDataIn | LyDataOut | 備註 |
|---|---|:--:|:--:|---|
| `000000` | 貨品基本資料 | ✔ | ✔ | 主檔；`SK_*`（庫存推送用 `SK_NOWQTY`/`SK_STOP`/`SK_RKWHNO`）|
| `000001` / `00000D` | 客戶基本資料 | ✔ | ✔（用 `00000D`）| `CT_*` |
| `000004` | 倉庫基本資料 | ✔ | ✔ | `WH_*` |
| `000009` | 目前庫存（廠內倉）| — | ✔ | 逐倉別在庫：`WH_NO`/`WH_NAME`/`@WD_AMT`(目前庫存)/`@WD_AMTOK`(可用量)|
| `0000A0` | 訂貨單 | ✔ | ✔ | 客戶訂單（非庫存異動）|
| `0000A1` | 銷貨單 | ✔ | ✔ | **出庫**；主 `SP_`／明細 `SD_` |
| `0000A2` | 銷貨退回單 | ✔ | ✔ | **入庫**；主 `SP_`／明細 `SD_`（欄位同銷貨單）|

WCF 函式：`LyGetPassKey`（取金鑰，30 秒有效）/ `LyDataIn`（寫）/ `LyDataOut`（讀）/
`LyDataPage`（分頁）/ `LyDataDel`（刪）。測試主機 `http://api.lyserp.com.tw/erpapi/erpservice.svc`。

### 3.1 銷貨單／銷退單 常用欄位（0000A1、0000A2 共用前綴 SP_/SD_）

**主表（抬頭）`SP_`**：`SP_NO` 單據編號、`SP_DATE` 貨單日期、`SP_CTNO` 客戶編號、
`SP_CTNAME` 客戶名稱、`SP_DPNO` 部門編號、`SP_CHECK`/`SP_CHECK2` 是否確認/審核、`SP_REM` 備註。
（銷退單 `SP_SNO`＝對應的原銷貨單號。）

**明細 `SD_`**：`SD_SEQ` 明細序號、`SD_DATE` 貨單日期、`SD_NO` 貨單編號、`SD_CTNO` 客戶編號、
`SD_SKNO` **貨品編號（查詢條件）**、`SD_NAME` 品名、`SD_UNIT` 單位、`SD_QTY` 驗收數量、
`SD_PRICE` 單價、`SD_STOT` 小計、`SD_WHNO` 倉庫(入)、`SD_WHNO2` 倉庫(出)、`SD_PH` 批號。

### 3.2 貨品主檔 000000 常用欄位

`SK_NO` 商品編號、`SK_NAME` 品名、`SK_SPEC` 規格、`SK_UNIT` 基本單位、`SK_RKWHNO` 入庫倉庫、
`SK_STOP` 是否停用（1=停用，推送時過濾）、`SK_KCKC` 是否計庫存、`SK_USEPH` 使用批號、
`SK_UPDATE1` 最近入庫日、`SK_UPDATE2` 最近出庫日。
（`SK_NOWQTY` 現有量為即時計算欄，庫存推送 `ly_stock_push.py` 用。）

### 3.3 LyDataOut 查詢重點（踩過的坑）

- **明細查詢**：`idetfields` 帶明細欄位、`irwhere` 用 `SD_SKNO=@v1@`、`iwhval` 給料號；
  一次回主表 `LYDATATITLE` ＋ 明細 `LYDATADETAIL`，本地用 `SD_NO==SP_NO` 對映。
- **過濾式值多個**用 `@#1#@` 連接；`iorder` 要整句（如 `order by SP_NO desc`）。
- **審核狀態**：新寫入常 `SP_CHECK=0`（未審核）；很多凌越畫面預設只顯示已審核 →
  查詢時**不要**加 `SP_CHECK=1` 的條件，否則會漏抓剛開的單。
- **公司別 ICPNO**：松富＝`00`（或測試庫 `108`）；查不到八成是公司別不對。

## 4. 日後若凌越「有」進貨單，怎麼開啟

這份 API 目前沒有進貨單。若之後向凌越（Dispatch）取得進貨單的資料種類代碼：

1. 在「凌越整合代理」的環境變數（或視窗⚙設定）加：
   ```
   LY_PURCHASE_IDAKD=<進貨單代碼>       # 例如 0000B1（實際以凌越提供為準）
   LY_PURCHASE_PREFIX=SP_/SD_          # 進貨單主表/明細前綴；不同就改，如 PP_/PD_
   ```
2. 重開視窗即生效：進銷交易會多查一種、標成**入庫(＋)**、類型顯示「進貨」。
3. 不必改雲端、不必改 `ly_item_txn.py`（已內建這個掛勾）。

> 驗證：內網那台可先單獨測 `python ly_item_txn.py --code <料號> --purchase-kind <代碼>`，
> 看有沒有撈到進貨明細、方向對不對，再設環境變數上線。

## 5. 驗證

```bash
# 內網那台（連得到凌越）單獨測某料號的近期進銷交易：
python ly_item_txn.py --code 10100004 --icpno 00
#   ▶ 料號 ... 近期進銷交易：N 筆  Σ入 .. / Σ出 .. / 淨 ..
#   （銷貨=出庫-、銷退=入庫+）

# 雲端：庫存管理 → 目前庫存 → 點任一品項 → 右側抽屜出現 Σ入/Σ出/淨變動 + 明細
#   需「凌越整合代理」視窗常駐（_txn_loop 長連線）才會即時回填。
```
