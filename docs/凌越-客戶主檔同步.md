# 凌越客戶主檔同步（erp_customers）

最後更新：2026-07-11

## 這是什麼

把凌越 ERP 的**客戶主檔（資料種類 `000001`）**整批同步到雲端後台，
後台「客戶管理 → 凌越客戶主檔」直接看得到最完整的客戶資料（地址、電話、統編、
負責人、付款方式、業務員…），不用再手動抄進網站。

- **唯讀快照**：資料權威在凌越。要改客戶資料請在凌越改，下次推送自動更新。
- **隨庫存推送自動同步**：內網代理每次推庫存（排程或網站按「庫存更新」）時順帶推客戶主檔，
  不需要新的排程或新的常駐執行緒。
- **多公司**：與庫存一樣按公司（icpno）逐家推、按公司覆蓋，互不影響。

## 資料流

```
凌越 (LAN)                內網 Windows                          雲端 Cloud Run
客戶主檔 000001  ──lystk──▶ ly_stock_push.py                ──▶ POST /admin/lingyue-writeback/customers-push
                            （庫存推送成功後順帶推；                （X-Writeback-Key 認證；單一交易按 icpno
                              失敗只警告，不影響庫存）                DELETE+INSERT 覆蓋 erp_customers）
```

- 雲端資料表 `erp_customers`，主鍵 **(icpno, ctno)**；常用欄位攤平＋
  **`raw_json` 存整筆原始欄位**（凌越有的欄位全保留，之後要顯示什麼都查得到）。
- 快照時間/筆數記在 `app_settings`：`erp_customers_snapshot_at_<icpno>`、`erp_customers_count_<icpno>`。
- **停用客戶不過濾**（與庫存推送的 SK_STOP 過濾不同）：後台要看最完整資料，
  `stop` 欄位帶上去由頁面淡化標示「停用」。

## 欄位對映（防禦式候選偵測）

已知欄位只有 `CT_NO / CT_NAME / CT_UNINO / CT_ADDR1 / CT_FKFS / CT_SALES`
（見 `docs/凌越串接-通用方法說明.md`），其餘欄名是常見猜測，
`ly_stock_push.py` 的 `CUST_FIELDS` 用候選清單逐一偵測（第一列偵測、整批套用），
**偵測不到就留空、不會炸**；不管攤平成不成功，整筆原始欄位都進 `raw_json`。

| 雲端欄位 | 候選凌越欄名 |
|---|---|
| ctno / name | `CT_NO` / `CT_NAME`（必要；CT_NO 偵測不到＝本輪不推） |
| short_name | `CT_SNAME`, `CT_ABBR`, `CT_SHORTNAME` |
| addr1 / addr2 | `CT_ADDR1`(`CT_ADDR`) / `CT_ADDR2` |
| tel1 / tel2 / fax | `CT_TEL1`(`CT_TEL`) / `CT_TEL2` / `CT_FAX`(`CT_FAX1`) |
| unino（統編） | `CT_UNINO` |
| boss / contact | `CT_BOSS`(`CT_OWNER`) / `CT_LINKMAN`(`CT_CONTACT`,`CT_LINK`) |
| fkfs / sales | `CT_FKFS` / `CT_SALES` |
| stop | `CT_STOP`（1/Y=停用） |

第一次實跑前先在內網 `python ly_stock_push.py --dry-run --icpno 00` 看
`[000001]` 原始樣本，欄名對不上就補進 `CUST_FIELDS` 候選清單（換 .py 即生效）。

## 後台入口

- **客戶管理 → 凌越客戶主檔**（`/admin/customers/erp`）：公司切換、搜尋
  （編號/名稱/地址/電話/統編）、與網站客戶比對（`customers.hq_cust_code`＝凌越 `CT_NO`）。
  - 已對應 → 點名稱進該客戶編輯頁。
  - 未對應 → 「建立客戶」一鍵帶名稱/凌越編號/電話到新增客戶頁，確認即建檔
    （**這就是「不用一直手動新增資料」的入口**）。
- **客戶管理 → 編輯客戶**：有填凌越編號（HQCustCode）的客戶會多一張
  「凌越主檔資料（自動同步，唯讀）」卡，顯示地址/電話/統編/付款/業務等；
  「帶入聯絡方式」按鈕把凌越電話填進網站聯絡欄（要按儲存才寫入）。

## 上線步驟（內網）

1. 把 repo 最新的 `scripts/ly_stock_push.py` 覆蓋到內網「凌越整合代理」資料夾
   （exe 同層，`local_import` 會優先撿走，**不必重打包**）。
2. 先 `python ly_stock_push.py --dry-run --icpno 00` 核對 `[000001]` 欄位樣本。
3. 之後每次庫存推送（排程／網站按「庫存更新」）都會自動同步客戶主檔；
   後台「客戶管理 → 凌越客戶主檔」看得到最後同步時間。

## 與「訂單回寫沒有客戶地址」的關係

這條同步線解決的是**網站端看得到完整客戶資料**；至於寫進凌越的訂貨單本身
沒帶地址欄位（凌越 UI 選客戶會自動帶、API 寫入要自己補，同 OR_FKFS/OR_SALES 的教訓），
那要在 `ly_writeback_bridge.py` 的 `map_order()` 補訂貨單地址欄位——
訂貨單主表的地址欄位確切名稱待內網查一張手打舊單確認後再補（見該檔 `customer_defaults()`）。
