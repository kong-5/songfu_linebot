# 凌越訂單回寫 — 規則與必備設定（權威版）

> 這份是「不要再重複踩」的清單。整合代理（`ly_agent_gui.py`）與 CLI 橋接
> （`ly_writeback_bridge.py`）都依這份規則。**改規則就改這份 + 對應程式。**

---

## 0. 最重要的一條：什麼時候才會寫進凌越？

**只有你在網站點『上傳凌越』的那幾張才會寫。** 其他訂單一律不碰。

- 網站點「上傳凌越」→ 後台把該筆**排入佇列**（`lingyue_queued_at`）。
- 內網代理**長連線等 `/admin/lingyue-writeback/wait`**，這支**只回你排入佇列的單**。
- 代理寫入凌越後，用 `/callback` 回填單號，該筆即退出佇列。

❌ **絕對不可**用 `/pending`（那是「當日全部未回寫」的清單）。
之前 2026-07-08 誤把 60 張整天訂單全寫進凌越（單號 202607080064–202607080123），
就是因為代理接了 `/pending` 又每 10 分鐘盲掃。**已改為只接 `/wait`。**

## 1. 必備設定（缺一不可）

| 設定（視窗⚙）| 環境變數 | 必填 | 說明 |
|---|---|:--:|---|
| 雲端後台網址 | `LY_CLOUD_BASE` | ✅ | 例 `https://xxxx.run.app` |
| 回寫金鑰 | `LINGYUE_WRITEBACK_KEY` | ✅ | 與後台環境變數相同那把 |
| 公司代碼 | `LY_ICPNO` | ✅ | 松富=00、龍港=01、桂田=03 |
| **預設倉別** | `LY_DEFAULT_WHNO` | ✅ | **一定要填**，否則匯入的訂單沒有倉別（見 §3） |
| 預設單價 | `LY_DEFAULT_PRICE` | ⬜ | 留空＝讓凌越依客戶售價表帶價（建議留空） |

> 倉別留空時，程式會在記錄框跳警告「寫入凌越的訂單會沒有倉別」。

## 2. 欄位對映規則（雲端訂單 → 凌越訂貨單 OR_/OD_）

| 凌越欄位 | 來源 | 規則 |
|---|---|---|
| `OR_CTNO` | customer_code | 凌越客戶編號（HQCustCode，無則寺岡碼） |
| `OR_CTNAME` | customer_name | |
| `OR_DATE1/2` | order_date | 正規化 `YYYY-MM-DD` |
| `OR_REM` | doc_remark | |
| `OR_CHECK` | — | 固定 `0`（不審核，方便需要時刪） |
| `OD_SKNO` | product_code | 凌越料號（erp_code）。**空料號的品項會被跳過**（見 §4） |
| `OD_NAME` | product_name | |
| `OD_UNIT` | unit | **單位正規化：公斤 → KG**（見 §5） |
| `OD_WARE` | 設定的倉別 | 由 `LY_DEFAULT_WHNO` 帶入（見 §3） |
| `OD_QTY` | quantity | |
| `OD_PRICE` | 設定的單價 | 留空＝不送，讓凌越帶價 |
| `OD_REM` | item_note | 有備註才送 |

## 3. 倉別規則（OD_WARE）— 待你確認

**現況**：預設留空 → 匯入的訂單沒有倉別。**這是要修的。**

**做法（三選一，請你決定）**：
- **A. 全部固定用同一個倉別**：設 `LY_DEFAULT_WHNO=FN005`（或你的實際出貨倉代碼）。最簡單。
- **B. 每個品項用凌越貨品主檔的預設倉別**（`SK_RKWHNO`，如 FN001/FN002/FN013）。最貼近實際，需程式多做一次「料號→倉別」對照查詢（可加）。
- **C. 留空、之後在凌越手動補**（＝目前狀況，通常不要）。

> ✍️ **決定後填在這裡**：目前採用＿＿＿＿；固定倉別代碼＝＿＿＿＿。

---

## 4. 缺凌越料號的品項會被「跳過」

凌越不收沒有料號的明細行。程式遇到**沒有 `erp_code`（凌越料號）**的品項會：

- **跳過該品項**（不寫該行），並在記錄框警示，例如
  `⚠ 跳過無凌越料號品項：梅乾菜（客戶 …）`。
- 其餘有料號的品項照常寫入，整張單不會因此失敗。

👉 **要讓這些品項也進凌越**：到後台把該商品的**凌越料號（erp_code）補上**，
下次上傳就會帶進去。（常見缺料號：梅乾菜、箭筍、沙拉筍、麻竹筍、酒精膏…等）

---

## 5. 單位正規化規則（OD_UNIT）

| 雲端單位 | 寫入凌越 |
|---|---|
| 公斤 / kg / KG | **KG** |
| （空白） | KG |
| 其他（把、箱、盒、台斤…） | 原樣送出 |

> 對照表在 `ly_writeback_bridge.py` 的 `UNIT_MAP`。要再加（例如「台斤→?」）就改那張表。

## 6. 已驗證可用的操作步驟（SOP）

### A. 第一次設定
1. 解壓程式包（三個 `.py` 放同一資料夾），雙擊 `ly_agent_gui_run.bat`。
2. 「⚙ 設定」填 §1 的必備設定（**含倉別**）→「測試連線」顯示成功 → 儲存。

### B. 上線前先測一張（務必）
1. 到網站對**一張**訂單點「上傳凌越」。
2. 視窗按「**檢查上傳佇列(試跑)**」→ 只印不寫，確認品項/倉別/單位對。
3. 按「**立即處理上傳佇列**」→ 實際寫入那一張 → 到凌越確認：倉別有、單位 KG、料號對。

### C. 日常
- 讓代理掛著；你在網站點「上傳凌越」，代理數秒內就寫入並回填單號。
- 網站訂單列表會出現 `↩單據號`（青色）代表已寫入。

---

## 7. 出事了怎麼救（刪除誤寫的凌越單）

用 `ly_writeback_undo.py`（跑在內網那台）：

```bash
# 1) 先核對（不刪）：回查這批單號，印出日期/客戶
python ly_writeback_undo.py --range 202607080064 202607080123

# 2) 核對無誤後真的刪
python ly_writeback_undo.py --range 202607080064 202607080123 --confirm
```

- 刪掉凌越單後，後台那幾筆仍顯示「已轉入」。要重新上傳時，用網站該筆的
  「**↩ 重新轉入**」（它會清掉舊單號並重新排入佇列），代理就會再寫一次。

## 8. 端點速查（機器對機器，`X-Writeback-Key` 認證）

| 端點 | 用途 | 代理是否使用 |
|---|---|---|
| `GET /admin/lingyue-writeback/wait` | 長連線等「上傳凌越」佇列 | ✅ **訂單回寫只用這支** |
| `POST /admin/lingyue-writeback/callback` | 回填凌越單號 | ✅ |
| `GET /admin/lingyue-writeback/pending` | 當日全部未回寫（僅供連線測試/查詢） | 只用來「測試連線」，**不用來寫入** |
| `GET /admin/lingyue-writeback/inventory-wait` | 庫存即時刷新長連線 | 庫存用；舊版後台可能 404 |
| `POST /admin/lingyue-writeback/inventory-push` | 推目前庫存 | 庫存用 |

> 若 `/wait` 或 `/inventory-wait` 回 **404**，代表**線上後台版本較舊、缺這些端點**，
> 需要把後台**重新部署到最新版**才會有。
