## 1. 在做什麼（目標）

把松富後台的訂單**自動回寫／匯入凌越 ERP 銷貨單**，並把凌越配發的**單據號碼**顯示回後台方便核對。
取代現在「匯出 Excel → 手動匯入凌越」的流程。

## 2. 架構（為什麼分兩台）

凌越是**本機 LAN 版** SOAP API（`192.168.4.11`，不走外網）；後台跑在 **Cloud Run（外網）連不到內網**。所以拆兩段：

```
 你的 Mac                          公司 Windows (Dispatch 那台)
 ────────                          ──────────────────────
 npm run deploy                    py ly_writeback_bridge.py
   ├─ 部署後台程式 ──▶ Cloud Run ◀──── 抓 pending / 回填單號 (HTTPS)
   └─ 設 LINGYUE_WRITEBACK_KEY        └─ 寫入凌越 (LAN 192.168.4.11)
              ▲                                  ▲
              └────── 同一組金鑰（手動兩邊各貼一次）┘
```

- **Mac**：部署雲端後台、設金鑰。
- **公司 Windows**：跑橋接程式真正寫凌越（只有這台連得到凌越、有 `ly_datain.py`）。
- 兩台**不共用檔案**，唯一要手動搬的是**金鑰那串字**。

## 4. 已完成（都在 PR #1，已 push）

**雲端側（`dist/` 內，此專案維護編譯後的 dist）**
- DB 新增欄位：`orders.lingyue_doc_no`、`orders.lingyue_written_at`（SQLite + Postgres 遷移）
- `GET  /admin/lingyue-writeback/pending?date=YYYY-MM-DD` — 輸出該日未回寫訂單 JSON
- `POST /admin/lingyue-writeback/callback` — 回填凌越單號、記錄時間
- 認證：HTTP 標頭 `X-Writeback-Key` 需等於環境變數 `LINGYUE_WRITEBACK_KEY`
  （未設→`503`、不符→`401`、不走 cookie 登入）
- 後台顯示：訂單列表「↩單據號」標記、訂單明細標題顯示「凌越單據號 …」
- 已通過端到端煙霧測試（401 擋未授權／pending 正確／callback 回填／回填後不再出現／DB 確實寫入）

**內網側**
- `scripts/ly_writeback_bridge.py` — 拉 pending → 組 `write_invoice` row → 寫凌越 → 回填。
  三模式：`--dry-run`（只印）/ `--test`（寫一張測試單→驗證→刪）/ 正式整批。
- `scripts/ly_writeback_daily.bat` — Windows 工作排程器每日自動跑的包裝檔。
- `docs/凌越訂單回寫-串接說明.md` — 完整串接說明。

## 5. 欄位對映（已定案，對齊凌越 `ly_datain.write_invoice`）

| 凌越欄位 | 來源（雲端 pending） |
|---|---|
| `SP_CTNO` | `customer_code`（凌越編號 HQCustCode，無則寺岡碼） |
| `SP_CTNAME` | `customer_name` |
| `SP_DATE` | `order_date`（YYYY-MM-DD） |
| `SP_REM` | `doc_remark` |
| `SD_SKNO` | `product_code`（凌越料號 erp_code） |
| `SD_NAME` | `product_name` |
| `SD_UNIT` | `unit` |
| `SD_QTY` | `quantity` |
| `SD_REM` | `item_note` |
| `SD_WHNO` | **預設留空**（決定：之後在凌越補倉別） |
| `SD_PRICE` | **預設 0**（叫貨單無價；`SD_STOT` 由 ly_datain 自動算） |

- 單號 `SP_NO` 由 `ly_datain` 依「當日既有單據最大流水 +1」自動順編 → **天然不撞號**，雲端不指定。

## 8. 待釐清/決策中

- **倉別 SD_WHNO**：目前留空，等 `--test` 驗證凌越是否接受空倉別。若不接受 → 需決定「全部固定一個倉」或「品項對應不同倉（雲端要加欄位）」。
- **單價 SD_PRICE**：目前帶 0，使用者不確定凌越會不會有問題 → 同樣靠 `--test` 驗證。若要帶價，需接凌越價格表/客戶售價。

## 9. 重要提醒

- 金鑰兩邊**一字不差**：Cloud Run 的 `LINGYUE_WRITEBACK_KEY`（.env）＝ Windows 的 `$env:LY_WRITEBACK_KEY`。
- `npm run deploy` 的 `.env` 必須**完整**（含 LINE token、Gemini、DATABASE_URL 等所有正式變數），否則會把雲端其他環境變數洗掉。使用者既有 `.env` 本來就完整，加一行即可。
- 重複跑橋接程式**安全**：已回寫的訂單不會再出現在 pending，不會重複寫入凌越。
- 此專案版控的是編譯後的 `dist/`（無 `src/`），所以雲端邏輯改在 `dist/admin/index.js`、`dist/db/index.js`。
