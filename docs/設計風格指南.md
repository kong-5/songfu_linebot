# 設計風格指南

> 松富系統（LINE Bot 後台＋內網桌面工具）的視覺與文案慣例。
> 新功能、新視窗、新頁面照這份做，風格才會一致。

---

## 1. 通用原則

- **語言**：介面、記錄、文件一律**繁體中文**；欄位代碼保留原文（`OR_CHECK`、`SK_RKWHNO`）。
- **面向使用者的文案**講「人話」：說「上傳凌越」不說「POST /wait 佇列」；
  錯誤訊息要說**原因＋下一步**（例：「金鑰不正確（與後台 LINGYUE_WRITEBACK_KEY 不符），請按⚙設定修正」）。
- **狀態符號**（記錄/console 通用）：
  `▶` 開始　`✅` 成功　`❌` 失敗　`⚠` 警告　`ℹ` 說明　`🔄` 進行中
  `⏰` 定時　`📝` 收到工作　`🗑` 刪除　`↩` 回填/轉入　`⚪` 略過
- **危險操作預設安全**：先 dry-run 再 `--confirm`；先「試跑」再「立即處理」；
  預設值永遠選不會出事的那個。

## 2. 桌面工具（tkinter，「凌越整合代理」風格）

深色現代風，配色寫死在 `ly_agent_gui.py` 頂部，全部沿用：

| 用途 | 色碼 |
|---|---|
| 視窗底 `BG` | `#1b1d2a` |
| 卡片底 `PANEL` / 邊框 `PANEL2` | `#252838` / `#2d3145` |
| 主文字 `FG` / 次要 `MUTED` | `#e6e6e6` / `#9aa0b4` |
| 成功/運作 `GREEN` | `#3fb950` |
| 錯誤/停止 `RED` | `#f85149` |
| 警告 `AMBER` | `#d29922` |
| 資訊/主要動作 `BLUE` | `#58a6ff` |
| 停用 `GREY` | `#6b7280` |
| 記錄框底/字 | `#14151f` / `#c8d0e0` |

- **字型**：中文 UI＝`Microsoft JhengHei UI`（標題 bold 12–18、內文 10）；
  記錄框/代碼＝`Consolas 10`。
- **版面**：頂部標題列（左標題、右按鈕）→ 2×N 狀態卡 → 手動操作列 → 記錄框。
- **狀態卡**＝`●`圓點（顏色表狀態）＋粗體標題＋兩行小字（最後時間等）。
  加新模組＝加一張卡＋一個背景執行緒，不動其他區塊。
- **按鈕語意**：綠＝啟動/儲存/測試連線、紅＝停止、藍＝主要動作（立即推庫存）、
  琥珀＝會寫入 ERP 的動作（立即處理上傳佇列）、灰＝一般。
- **執行緒紀律**：背景工作絕不直接碰 UI——`print`/log 進 queue，
  主執行緒 `after()` 輪詢顯示；所有碰凌越的操作拿同一把 `erp_lock`。
- 設定視窗：分區標題用藍色粗體；欄位＝粗體標籤＋輸入框＋灰色小字提示；
  金鑰欄位遮罩＋「顯示金鑰」勾選；底部左「測試連線」右「取消/儲存」。

## 3. 後台網頁（admin）

- 狀態一律用**膠囊標籤（pill）**呈現，如已轉入凌越：
  `✓已轉入 ↩單號`（青綠底 `#e0f2f1`、深青字 `#00695c`、粗體）。
- 訂單對話視圖配色語意：**藍底＝已對應到明細、黃底＝客戶訊息、灰底＝同事回覆**。
- 會產生後果的按鈕要帶確認訊息，且訊息把後果講完整
  （例：重新轉入會產生新單、舊單不會自動刪）。
- 列表頁資訊密度優先：小徽章、緊湊行距；可點的標籤要看起來可點。

### 3.1 圖示（icon）— 一律「線條（line art）」，不用彩色 emoji ⚠必讀

- **UI 圖示一律用線條風格**：走 `dist/admin/index.js` 的 `SF_ICONS` 集
  （`viewBox 0 0 16 16`、`stroke="currentColor"`、`stroke-width 1.4`、`fill="none"`、無彩色）。
  顏色靠 `currentColor` 繼承，跟著文字走；**禁止**把彩色 emoji（📢📈📅📋📊👤🖼🛒…）當 UI 圖示。
- **怎麼用**：
  - flex 容器（如 `.sf-card-title`、`.sf-card-head`）內直接內嵌 `${SF_ICONS.鍵}`，靠 `gap` 排版。
  - 非 flex 容器（按鈕 `.sf-btn`、分頁 `.sf-tab`、行內標籤）用 `${sfInlineIcon('鍵')}`，
    會包 `<span class="sfi">` 幫你對齊。
- **現有鍵**（不夠再照同格式新增，不要臨時塞 emoji）：
  `bell check x plus search filter chev_r chev_d thermo box truck users list history warn dl
  refresh spark edit dots link sun moon menu megaphone bolt calendar clipboard note chartLine
  chartBar image printer message mail cart pin wand money user doc scale stop tag`。
- **新增圖示**＝在 `SF_ICONS` 加一條同規格 SVG（16px、1.4 stroke、currentColor、fill none），
  命名用語意英文小寫；**不要**改成填色或多色。
- **不在此限的三種既有符號**（保留、不要硬轉 SVG）：
  1. 送進 **LINE 訊息 / Flex 泡泡**的文字裡的 emoji（LINE 端無法渲染內嵌 SVG，且是給客戶看的）；
  2. **log／console／toast 狀態符號**：`▶ ✅ ❌ ⚠ ℹ 🔄 ⏰ 📝 🗑 ↩ ⚪`（見 §1，屬狀態語彙非 UI 圖示）；
  3. **內容語意標記**：畜產／雞蛋等品類 `🐖 🐓 🥚 🐄 🐟`（是資料標籤，不是介面圖示）。

### 3.2 版面寬度 — 頁面要「吃滿視窗」，別卡在窄欄 ⚠必讀

- 桌面常常「內容擠在中間、兩側一堆空白」＝版面太窄。**頁面要用滿可用視窗寬度**。
- **主內容區 `.notion-main`**：`max-width: min(100%, 1600px)`＋左右 `padding: clamp(24px, 3.5vw, 56px)`
  ——螢幕越寬留白越合理，但不再死卡在窄欄。
- **資料密集頁（表格、盤點、分析）**用 `.sf-root` / `body.sf-fullwidth` 直接滿版，
  不套 `.notion-main` 的寬度上限。
- 需要限制行寬以利閱讀的純文字區塊，才在**該區塊**上加 `max-width`，**不要**用整頁窄欄達成。
- 只有手機／平板（`≤1024`、`≤760`）才收窄、側欄轉抽屜；桌面一律吃滿。

### 3.3 滑桿元件（開關／分段）— 全站只有兩種，禁止再自刻 ⚠必讀

系統裡「滑桿類」控制項一律收斂成**兩個標準元件**，玻璃通透同一套視覺（藍 `#2383e2`）：

1. **開關（on/off）＝ `.sf-switch`** — 二元切換（隱藏 0、啟用此品項…）。
   ```html
   <label class="sf-switch-label"><input type="checkbox"><span class="sf-switch"></span>文字</label>
   ```
   底層是**真 checkbox**（表單／JS 直接讀 `.checked`）；關＝灰底、開＝藍底＋白鈕右滑。
2. **分段滑桿（choose one）＝ `.sf-seg`** — 在幾個檢視／模式間切一個（列表／依倉庫、價格／管理品項…）。
   ```html
   <div class="sf-seg"><button class="active">列表</button><button>依倉庫</button></div>
   ```
   導覽型（點了換頁）用 `<a class="on">…</a>`。玻璃通透底、**選中子項亮白膠囊**。
   JS 版切換時把 `.active` 換到被點的按鈕即可（膠囊自動跟著），不需自己畫滑塊。

- **怎麼選**：狀態是「是／否」→ 用**開關**；是「在多個視圖／模式選一個」→ 用**分段滑桿**；
  切換整個分頁內容的**頁面導覽**→ 用 `.sf-tab` 底線分頁（那是 tab，不是滑桿）。
- **禁止**再寫 `stk-seg`／`pe-switch`／`qe-seg` 這種一次性樣式；新切換一律套這兩個 class。
- **LIFF（手機頁）**沿用同一套視覺。**唯一例外**＝帶紅綠語意的**狀態鈕**
  （冷凍櫃「正常／異常」`freezer-temp.html` 的 `.seg`）：保留綠＝正常、紅＝異常，不套中性玻璃樣式。

## 4. Python 腳本（scripts/ly_*.py）

- 檔頭一律有**繁中 docstring**：用途、跑在哪台、流程、環境變數、用法範例。
- 命名：內網凌越工具一律 `ly_` 前綴；常駐叫 `agent`、單次叫 `push/query/write`。
- CLI 用 argparse＋繁中 help；預設值走環境變數（`LY_*`）、CLI 可覆蓋。
- 標準庫優先（urllib 而非 requests），減少內網機器裝套件。
- 啟動時 `sys.stdout.reconfigure(encoding="utf-8", errors="replace")` 防中文亂碼。
- 共用邏輯放 bridge（如 `map_order`）；其他程式 import 它，**不准複製貼上再改**。

## 5. 文件（docs/*.md）

- 檔名用中文、講內容（`資料處理規則.md`、`內網設定與部署.md`）。
- 開頭一段引言講「這份是什麼、跟誰相關」；多用表格；指令給完整可複製區塊。
- 重要教訓寫成「已知陷阱」清單留在文件裡，不讓同樣的坑踩第二次。
- 規則變更＝同一次 commit 改文件＋改程式，兩者不可分開。

## 6. Git

- Commit 訊息：`type(scope): 繁中摘要`（feat/fix/change/docs），內文條列做了什麼、為什麼。
- 一個 PR 一個主題；merge 用 squash 保持 main 線性。
- 內網腳本也一律進 repo（main）——不進版控的程式等於不存在。
