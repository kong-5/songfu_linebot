# 庫存管理 — 目前庫存（凌越 SK_NOWQTY）

後台「庫存管理 → 目前庫存」顯示凌越每個品項的**目前在庫量**，可每日自動刷新、也可點按鈕即時刷新，
支援列表檢視與「依倉庫」分組（凌越倉別 / 自訂庫房兩種），畫面緊湊、可搜尋、可匯出目前檢視。
最後更新：2026-07-06

---

## 1. 資料是什麼

- 來源＝凌越貨品主檔（資料種類 `000000`）的 **`SK_NOWQTY`（現有量）**，每品項一個「公司總量」。
  對照：料號 `SK_NO`、品名 `SK_NAME`、規格 `SK_SPEC`、單位 `SK_UNIT`、預設入庫倉別 `SK_RKWHNO`。
- **負值是正常的**：出貨/銷貨先入帳、進貨還沒補時，帳面現有量會是負數。
- 「依倉庫」只是把品項**歸到某個倉來分組**，不是把數量拆到各倉（凌越給的是單一總量）。
  - **凌越倉別**：用 `SK_RKWHNO`（如 FN001/FN002/FN013）分組，免設定。
  - **自訂庫房**：用後台「盤點作業 → 品項歸倉」的設定分組；未歸倉者列在「（未歸倉）」。

---

## 2. 架構（雲端連不到凌越 LAN，一律兩段）

```
 內網 Windows（連得到凌越）                         雲端後台 Cloud Run
 ─────────────────────────                         ──────────────────
  ly_stock_push.py  撈 SK_NOWQTY ──POST 整批──▶  /admin/lingyue-writeback/inventory-push
  ly_stock_agent.py long-poll   ──等刷新請求──▶  /admin/lingyue-writeback/inventory-wait
                                                    ▼ 存入 erp_stock_items（全表覆蓋）
                                 使用者看 ◀──────  /admin/inventory/stock（庫存管理→目前庫存）
```

- 機器端點沿用**訂單回寫同一把金鑰** `LINGYUE_WRITEBACK_KEY`（標頭 `X-Writeback-Key`），不必另設。
- 「庫存更新」按鈕：後台記一個待處理旗標 → 常駐的 `ly_stock_agent.py` long-poll 立刻撿走 → 撈凌越 → 推回 → 頁面自動重載。

---

## 3. 內網那台要跑什麼（D:\Work\lystk_tool）

三個檔（與 `lystk.py`、`ly_query_stock.py` 同目錄）：

| 檔案 | 作用 | 怎麼跑 |
|---|---|---|
| `ly_stock_push.py` | 撈庫存**推送一次** | 手動或被 .bat 呼叫；`python ly_stock_push.py --dry-run` 可先試 |
| `ly_stock_daily.bat` | **每日固定刷新**（保底） | 工作排程器：每日（可排多個時間）啟動此 .bat，開始位置 `D:\Work\lystk_tool` |
| `ly_stock_agent.py` | **即時刷新**（常駐 long-poll） | 工作排程器：登入時啟動＋失敗自動重啟；提供『按按鈕就刷新』 |

環境變數（兩支都讀）：

```
set LY_CLOUD_BASE=https://<後台網址>
set LY_WRITEBACK_KEY=<與後台 LINGYUE_WRITEBACK_KEY 相同>
set LY_ICPNO=00          # 松富；01=龍港、03=桂田
set LY_STOCK_TIMES=06:00,12:00   # 代理每天自動推送的時間；清空＝只靠按鈕觸發
```

> 只想要「每日固定刷新」→ 只掛 `ly_stock_daily.bat` 即可。
> 想要「點按鈕即時刷新」→ 需另外**常駐**跑 `ly_stock_agent.py`（沒常駐時按鈕會等到逾時，並提示代理未執行；請求會記著，代理下次上線補刷新）。

---

## 4. 雲端只需確認一件事

`LINGYUE_WRITEBACK_KEY` 已設（訂單回寫在用就已經有）。沒有的話：

```bash
gcloud run services update songfu-line-bot --region asia-east1 \
  --update-env-vars "LINGYUE_WRITEBACK_KEY=<金鑰>"
```

資料表 `erp_stock_items` 由程式啟動時自動建立（SQLite/Postgres 皆有），免手動遷移。

---

## 5. 驗證

```bash
# 內網：先 dry-run 看撈得到幾筆
python ly_stock_push.py --dry-run
# 內網：正式推一次
python ly_stock_push.py
# 雲端：打開「庫存管理 → 目前庫存」應看到品項與「資料時間」；按「庫存更新」→ 需 agent 常駐才會即時刷新
```
