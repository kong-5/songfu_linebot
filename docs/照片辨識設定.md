# 上傳照片沒有辨識？請設定 Google Cloud Vision API

若 **LINE 傳照片** 或 **訂單整理上傳圖片** 都沒有辨識出文字，通常是未設定或未啟用 **Google Cloud Vision API**。

---

## 1. 確認環境變數

### 本機開發

在專案目錄（songfu_linebot）建立或編輯 **`.env`** 檔案，加入一行：

```bash
GOOGLE_CLOUD_VISION_API_KEY=你的Vision_API金鑰
```

存檔後**重新啟動**本機服務（`npm start`），即可在本機測試照片辨識。金鑰取得方式同下方「啟用 Cloud Vision API」與「取得 API 金鑰」。

### Cloud Run

在 **Cloud Run** → 您的服務 → **變數與密碼** 中，需有：

- **名稱**：`GOOGLE_CLOUD_VISION_API_KEY`
- **值**：Google Cloud **Vision API** 用的 API 金鑰（不是 Gemini、不是 OAuth，是「API 金鑰」）

若沒有這個變數，或填成別的金鑰（例如 Gemini），照片就不會辨識。

---

## 2. 啟用 Cloud Vision API

1. 開啟 [Google Cloud Console](https://console.cloud.google.com/)
2. 選對專案（與 Cloud Run 同一個）
3. 左側 **「API 和服務」** → **「已啟用的 API 和服務」**（或「程式庫」）
4. 搜尋 **「Cloud Vision API」**
5. 點進去後按 **「啟用」**

未啟用時，即使用對金鑰也會回傳錯誤、辨識不到。

---

## 3. 取得 API 金鑰（若還沒有）

1. **API 和服務** → **憑證** → **「建立憑證」** → **「API 金鑰」**
2. 建立後可選擇 **「限制金鑰」**：應用程式限制選「無」或「HTTP 參照」；API 限制可只勾選 **「Cloud Vision API」**
3. 複製金鑰字串，貼到 Cloud Run 的 `GOOGLE_CLOUD_VISION_API_KEY`

---

## 4. 區分三種金鑰（不要混用）

| 變數名稱 | 用途 | 取得處 |
|----------|------|--------|
| `GOOGLE_CLOUD_VISION_API_KEY` | **照片 OCR 辨識**（LINE 收圖、訂單整理上傳圖片） | GCP 憑證 → API 金鑰，且需啟用 **Cloud Vision API** |
| `GOOGLE_GEMINI_API_KEY` | 訂單明細「AI 分析」按鈕 | [Google AI Studio](https://aistudio.google.com/app/apikey) |
| `LINE_CHANNEL_ACCESS_TOKEN` | LINE 機器人發訊、收訊 | LINE Developers Console |

照片辨識**只用** Vision API 金鑰，不要用 Gemini 金鑰代替。

---

## 5. 設定完成後

- **LINE**：在群組傳一張有文字的訂單照片，應會回覆「已收到圖片並辨識出 N 項」或「已開始收單並辨識出 N 項」（有辨識到品項時）。
- **訂單整理**：在「新增紙本訂單」上傳圖片後按「以 Google AI 辨識」，應會出現辨識出的文字與品項預覽。

若仍無辨識結果，請看 **Cloud Run 日誌** 是否有 `[vision-ocr] API 回應非 200` 或 `403`，依錯誤訊息檢查金鑰與 API 是否啟用。
